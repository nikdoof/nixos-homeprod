    mx-01: Mail Server

    mx-01 is a NixOS microVM (CID 12) on hyp-01, running on VLAN 106 with a publicly routed IP address. It handles inbound and outbound email for two domains using Postfix (MTA) and Dovecot (IMAP/LMTP), with DKIM signing, DMARC policy enforcement, and SPF validation.

    Network

    Property Value
    Platform microVM, CID 12, VLAN 106
    IPv4 217.169.25.11/29
    IPv6 2001:8b0:bd9:106::3/64
    Gateway 217.169.25.9 (gw hosted VLAN)
    DNS 217.169.25.9, 2001:8b0:bd9:106::1 (Unbound on gw)

    Open TCP ports: 25 (SMTP/postscreen), 465 (SMTPS submission), 587 (submission), 993 (IMAPS).

    Hosted domains

    Virtual mailboxes are served for two domains:

    • doofnet.uk
    • dimension.sh

    The host's own FQDN (mx-01.doofnet.uk) is the only entry in mydestination. All mail for the virtual domains is delivered via LMTP to Dovecot.

    TLS / ACME

    A single TLS certificate covers mx-01.doofnet.uk, obtained via ACME DNS-01 challenge using the DigitalOcean API. After renewal, ACLs are applied so that dovecot2 and postfix can read the certificate files without running as acme, and both services plus opendmarc are reloaded. The certificate is used by both Postfix (inbound and outbound) and Dovecot (IMAP).

    Postfix

    Listeners

    Port Service Auth required Notes
    25 postscreen No Pre-screens before reaching smtpd
    587 submission Yes (SASL) TLS enforced (encrypt)
    465 submissions Yes (SASL) TLS wrapper mode

    Port 25 is handled by postscreen (not smtpd directly). The smtp_inet master.cf entry is overridden to run postscreen, which passes clean connections to a smtpd pass service. The dnsblog and tlsproxy helper processes are declared to support postscreen DNSBL lookups.

    Relay policy

    permit_mynetworks → permit_sasl_authenticated → reject_unauth_destination

    mynetworks covers loopback, the private VLAN (10.101.0.0/16), and the hosted /29 and /64, meaning infrastructure hosts can relay without authentication.

    Inbound TLS

    • Security level: may (opportunistic; enforced for submission ports)
    • Minimum protocol: TLSv1.2
    • Session cache: btree:/var/lib/postfix/data/smtpd_scache
    • smtpd_tls_received_header = yes — adds Received: trace with TLS info

    Outbound TLS (DANE)

    Outbound delivery uses smtp_tls_security_level = dane with smtp_dns_support_level = dnssec. For receiving servers that publish TLSA records, Postfix performs DANE-verified TLS; for those that don't, it falls back to opportunistic TLS. The DNSSEC-validating resolvers on mx-01 (Unbound on gw) are essential for this to work.

    Postscreen

    Postscreen pre-screens inbound SMTP connections before they consume smtpd resources:

    Check Action Detail
    DNSBL threshold ≥ 2 enforce zen.spamhaus.org*2, dnsbl.dronebl.org*2, bl.spamcop.net*1
    Bare newline enforce Guards against request smuggling
    Greeting banner timing enforce Rejects clients that pipeline before banner
    Blacklist action drop Silent drop for blacklisted IPs
    permit_mynetworks pass Skips all postscreen checks for trusted networks

    Passed connections are cached in btree:/var/lib/postfix/data/postscreen_cache.

    HELO, sender, and recipient restrictions

    HELO restrictions (in order):

    1. permit_mynetworks
    2. reject_non_fqdn_helo_hostname
    3. reject_invalid_helo_hostname
    4. reject_rhsbl_helo dbl.spamhaus.org
    5. permit

    Sender restrictions (in order):

    1. permit_mynetworks, permit_sasl_authenticated
    2. reject_non_fqdn_sender, reject_unknown_sender_domain
    3. reject_rhsbl_sender dbl.spamhaus.org
    4. permit

    Recipient restrictions (in order):

    1. reject_invalid_hostname, reject_unknown_recipient_domain
    2. reject_unauth_pipelining, reject_multi_recipient_bounce
    3. permit_mynetworks
    4. reject_non_fqdn_recipient, reject_unauth_destination
    5. RBL checks: dnsbl.dronebl.org, zen.spamhaus.org, bl.spamcop.net, b.barracudacentral.org, dnsbl-1.uceprotect.net
    6. check_policy_service unix:private/policyd-spf (SPF)
    7. permit

    CVE-2023-51764 (SMTP request smuggling)

    smtpd_forbid_bare_newline = yes is set globally. mynetworks hosts are exempted via smtpd_forbid_bare_newline_exclusions.

    Rate limiting

    Per-client limits measured over a 60-second window:

    Limit Value
    Connection rate 10/min
    Message rate 100/min
    Recipient rate 100/min
    Auth attempt rate 10/min

    Bounce / error handling

    Setting Value Rationale
    bounce_queue_lifetime 1d Short bounce window reduces backscatter risk
    smtpd_soft_error_limit 3 Delay after 3 soft errors
    smtpd_hard_error_limit 10 Drop after 10 hard errors
    smtpd_error_sleep_time 1s Slow-down delay per error

    Milters

    Two milters are applied to both smtpd_milters and non_smtpd_milters:

    1. OpenDKIM — DKIM signing (outbound) and verification (inbound)
    2. OpenDMARC — DMARC policy enforcement

    milter_macro_daemon_name = ORIGINATING is set on submission ports so milters can distinguish outbound authenticated mail from inbound.

    milter_default_action = accept means a milter failure does not block delivery.

    Aliases

    Local system aliases:

    Alias Destination
    root root-mail@m.tensixtyone.com
    inbox paperless, household@williams.id
    nikdoof andy@williams.id
    salkunh jo@williams.id

    Virtual aliases forward root@ for internal subdomains to root-mail@m.tensixtyone.com.

    SPF — policyd-spf

    SPF checking is performed by python-policyd-spf, spawned as a Postfix policy service on unix:private/policyd-spf. Policy:

    Condition Action
    HELO: SPF not pass Reject
    Mail From: SPF fail (hard) Reject
    SPF PermError Pass
    SPF TempError Defer

    Softfails are logged but not rejected.

    DKIM — OpenDKIM

    OpenDKIM runs in sv mode (sign outbound + verify inbound). Configuration:

    Setting Value
    Key path /persist/opendkim/keys
    Selector SHA1 hash of mx-01.doofnet.uk (deterministic)
    Domains doofnet.uk,dimension.sh
    InternalHosts Same as mynetworks

    Keys are stored on the /persist virtiofs share so they survive NixOS rebuilds. The DKIM public key for the generated selector needs to be published as a DNS TXT record at <selector>._domainkey.<domain> for each virtual domain.

    DMARC — OpenDMARC (modules/doofnet/opendmarc/)

    OpenDMARC is implemented as a custom NixOS module (doofnet.opendmarc) since there is no upstream NixOS module. It manages the opendmarc systemd service, user/group creation, and config file generation.

    Current settings on mx-01:

    Setting Value Notes
    AuthservID mx-01.doofnet.uk Identity added to Authentication-Results
    TrustedAuthservIDs mx-01.doofnet.uk Only trust results from self
    RejectFailures false Monitoring mode — logs but does not reject
    IgnoreAuthenticatedClients true Skip DMARC checks on authenticated submissions

    The RejectFailures = false setting means mx-01 is currently in DMARC monitoring mode. When SPF and DKIM are confirmed working correctly it should be set to true.

    The module runs OpenDMARC as the postfix user/group (shared with the Postfix process) so the milter socket is accessible.

    Dovecot

    Dovecot provides IMAP access and LMTP delivery for Postfix. POP3 is disabled. PAM authentication is disabled.

    Virtual users

    All mailboxes are virtual users authenticated against an passwd-file stored in an age-encrypted secret (secrets/mx01DovecotPasswd.age). Usernames are normalised to lowercase full email addresses (auth_username_format = %Lu).

    Mail is stored under: /persist/vmail/<domain>/<user>/Maildir/

    The vmail system user owns all mailbox files. On service start, an ExecStartPre step chowns the entire mail store to vmail:vmail.

    Mailboxes

    The following IMAP special-use folders are auto-created for all users:

    Folder RFC role Auto-expunge
    All \All Never
    Archive \Archive Never
    Drafts \Drafts Never
    Flagged \Flagged Never
    Junk \Junk 60 days
    Sent \Sent Never
    Trash \Trash 60 days

    LMTP socket

    Dovecot exposes an LMTP listener at /var/spool/postfix/dovecot-lmtp (inside the Postfix chroot), owned postfix:postfix. Postfix delivers via lmtp:unix:/var/spool/postfix/dovecot-lmtp.

    Authentication socket

    The SASL auth socket is at /var/spool/postfix/auth, owned postfix:postfix. Postfix uses this for both submission port SASL and inbound smtpd SASL.

    TLS

    Minimum TLS version: TLSv1.2. Server cipher preference is enabled.

    Metrics endpoint

    Dovecot exposes an HTTP stats endpoint on port 9166 (service stats). The following metrics are collected:

    Metric Filter Grouped by
    auth_success auth_request_finished, success=yes —
    imap_command imap_command_finished cmd_name, tagged_reply_state
    smtp_command smtp_server_command_finished cmd_name, status_code, duration histogram
    mail_delivery mail_delivery_finished duration histogram

    Fail2ban

    The dovecot jail is enabled via doofnet.fail2ban.jails.dovecot = true, banning IPs that exceed failed IMAP authentication thresholds.

    Persistence

    mx-01 uses the standard microVM /persist virtiofs share for state that must survive rebuilds:

    Path Contents
    /persist/vmail/ All user mailboxes (Maildir)
    /persist/opendkim/keys/ DKIM private keys
    /persist/ssh_host_* SSH host keys (standard microVM)
    /persist/acme/ ACME certificates

    Observability

    Three exporters run locally and are scraped by Alloy:

    Exporter Port Job Metrics
    prometheus-postfix-exporter 9154 postfix Queue lengths, delivery rates, error counts (systemd journal)
    Dovecot stats HTTP 9166 dovecot Auth, IMAP command, SMTP command, delivery metrics
    prometheus-dmarc-exporter auto dmarc DMARC report pass/fail counts (reads IMAP mailbox dmarc-reports@doofnet.uk)

    The DMARC exporter connects to Dovecot via IMAP using credentials loaded as a systemd credential file (compatible with DynamicUser), reading aggregate DMARC reports delivered to the dmarc-reports@doofnet.uk mailbox.

    DNS records required

    The following DNS records must be published for correct mail operation:

    Zone Name Type Value / Notes
    doofnet.uk @ MX mx-01.doofnet.uk
    dimension.sh @ MX mx-01.doofnet.uk
    doofnet.uk @ TXT SPF record (v=spf1 mx ~all or similar)
    dimension.sh @ TXT SPF record
    doofnet.uk <selector>._domainkey TXT DKIM public key (selector = SHA1 of mx-01.doofnet.uk)
    dimension.sh <selector>._domainkey TXT DKIM public key
    doofnet.uk _dmarc TXT v=DMARC1; p=none; rua=mailto:dmarc-reports@doofnet.uk (monitoring mode)
    dimension.sh _dmarc TXT DMARC policy
    Reverse 11 in 8-15.25.169.217.in-addr.arpa PTR mx-01.doofnet.uk.

    The DKIM selector is derived at build time as the SHA1 hash of the string mx-01.doofnet.uk and is therefore stable as long as the hostname does not change.
